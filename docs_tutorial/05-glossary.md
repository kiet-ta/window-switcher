# Standard Glossary

This glossary defines the technical terms utilized throughout our Hyprland Window Switcher tutorial.

- **Asynchronous (Async):** Things happening without stopping other things. Instead of waiting in a fixed line, the computer can do another task while waiting for the first task to finish. (Basic)
- **Channel:** A simple pipe or tube that allows two computer workers (threads) to send messages to each other securely. (Basic)
- **Compositor:** The supervisor that controls what you see. It draws all the app screens, animations, and shadows on your monitor. (Basic)
- **Concurrent Dispatch:** The simultaneous orchestration of multiple distinct functions across disparate CPU threads, inherently alleviating latency bottlenecks within heavy I/O operations. (Challenging / B2)
- **Glassmorphism:** A modern style for drawing boxes on a screen. It makes things look like blurry glass. (Basic)
- **GTK4:** A set of tools used to draw nice buttons, texts, and boxes on Linux. (Basic)
- **IPC (Inter-Process Communication):** The mechanism by which self-sufficient software binaries exchange serialized data sequences through a localized bridging Unix Socket. (Challenging / B2)
- **Layer Shell:** A special rule in Wayland that lets popups stick completely to the glass of the screen. Standard apps can never push these panels out of the way. (Basic)
- **MPSC (Multi-Producer, Single-Consumer):** A type of channel pattern. Many different workers can throw their notes into the same pipe, but only one boss reading at the very end receives all of them. (Basic)
- **Snapshot:** A picture taken of your screen. (Basic)
- **Tmpfs (Temporary File System):** A folder that does not use a real hard drive. It utilizes RAM (rapid system memory). Everything inside this folder vanishes indefinitely the exact moment the computer powers down. (Challenging / B2)
- **Wayland:** The new, improved boss of the Linux graphic system. It makes sure windows don't mess up how other windows look. (Basic)
